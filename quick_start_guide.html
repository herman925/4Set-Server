1 OneDrive Setup & Sync

Set up OneDrive for Business to sync your assessment files automatically. This is essential for the upload process.

Step 1.1: Install OneDrive for Business

  1. Open your web browser and go to office.com
  2. Sign in with your EdUHK account credentials
  3. Click the App Launcher (grid icon) in the top-left corner
  4. Select OneDrive
  5. Click Download at the top to install the OneDrive sync client
[Screenshot: OneDrive download page showing the Download button]

Step 1.2: Sign In to OneDrive

  1. After installation, open OneDrive from the Start menu
  2. Click Sign in
  3. Enter your EdUHK email address
  4. Enter your password when prompted
  5. Click Next and follow the setup wizard
[Screenshot: OneDrive sign-in screen]

Step 1.3: Configure Sync Folder

  1. During setup, select which folders to sync
  2. Find and check 4Set-Server folder
  3. Click Next to start syncing
  4. Wait for the initial sync to complete (may take several minutes)

‚úÖ Success! You should now see a green checkmark on the OneDrive icon in your system tray, indicating files are synced.

Step 1.4: Locate Your Sync Folder

  1. Open File Explorer (Windows key + E)
  2. Look for OneDrive - The Education University of Hong Kong in the left sidebar
  3. Navigate to the 4Set-Server folder
  4. Bookmark this location for easy access
[Screenshot: File Explorer showing OneDrive folder structure with 4Set-Server highlighted]

üí° Tip: Right-click the 4Set-Server folder and select Pin to Quick Access for faster navigation.

2 Uploading PDFs

Upload assessment PDF files following the correct naming convention. The system will automatically process them.

Step 2.1: Prepare Your PDF Files

‚ö†Ô∏è Important: File Naming Convention

Files must be named exactly like this:

xxxxx_YYYYMMDD_HH_MM.pdf

  • ‚Ä¢ xxxxx = 5-digit student ID (e.g., 13268)
  • ‚Ä¢ YYYYMMDD = Date (e.g., 20250904 for Sept 4, 2025)
  • ‚Ä¢ HH_MM = Time in 24-hour format (e.g., 14_07 for 2:07 PM)

Example: 13268_20250904_14_07.pdf

  1. Scan or export your assessment forms as PDF files
  2. Rename each file using the correct naming convention shown above
  3. Double-check the naming before uploading

Step 2.2: Upload via Web Interface

  1. Open the 4Set System in your web browser
  2. Click Assessment Uploader button on the home page
  3. On the upload page, configure PC Number (one-time setup)
  4. Set your Upload Destination to the OneDrive sync folder's incoming subfolder
  5. Drag and drop PDF files into the upload zone, or click to browse
  6. Wait for the upload progress bar to complete
[Screenshot: Assessment Uploader interface showing drag-and-drop zone]

üí° Alternative Method: You can also copy PDF files directly to the incoming folder in File Explorer instead of using the web interface.

Step 2.3: Verify Upload Success

After uploading, your files go through automatic processing:

  1. Files are moved from incoming to processing folder
  2. System validates the filename format
  3. PDF is parsed to extract data
  4. Data is uploaded to JotForm database
  5. Successfully processed files are moved to filed/S###/ (by school ID)
  6. Failed files are moved to filed/Unsorted/ for review

‚úÖ How to Check: Look in the filed/S###/ folder (where ### is the school ID). If your PDF and JSON files are there, upload was successful!

Step 2.4: What If Upload Fails?

Check the filed/Unsorted/ folder for:

  • Files with incorrect naming format
  • Files with validation errors (student not found, school mismatch)
  • Files that failed to upload to JotForm

To retry:

  1. Fix the issue (rename file, verify student ID, etc.)
  2. Move the PDF file back to the incoming folder
  3. System will automatically retry processing

3 Checking Upload Logs

Monitor the system's processing activity and troubleshoot issues using the log files.

Step 3.1: Locate Log Files

  1. Open the 4Set-Server folder in File Explorer
  2. Navigate to the logs subfolder
  3. Log files are named by date: YYYYMMDD_processing_agent.csv
  4. Open the most recent log file with Excel or a text editor
[Screenshot: Logs folder showing dated log files]

Step 3.2: Understanding Log Entries

Each log entry contains:

  • Timestamp - When the event occurred
  • Level - INFO, WARN, or ERROR
  • Filename - The PDF being processed
  • Message - What happened

üí° Common Log Messages

  • ‚Ä¢ "Jotform upload successful" - File processed successfully ‚úÖ
  • ‚Ä¢ "rejected" - Validation failed, check filename or data ‚ö†Ô∏è
  • ‚Ä¢ "PERMANENTLY FAILED" - Upload failed after retries ‚ùå
  • ‚Ä¢ "Filed to S###" - Successfully archived by school ID üìÅ

Step 3.3: Search for Specific Files

To find information about a specific upload:

  1. Open the log file in Excel
  2. Press Ctrl+F to open Find
  3. Search for the student ID or filename
  4. Review all matching entries to see the complete processing history
[Screenshot: Excel showing log file with search/filter applied]

4 Using the Checking System

The Checking System helps you monitor assessment completion and validate data quality at different levels.

Step 4.1: Access the Checking System

  1. Go to the 4Set System home page
  2. Click Checking System button
  3. Enter the system password when prompted
  4. You'll see the Checking System home page with multiple view options
[Screenshot: Checking System home page showing navigation options]

Step 4.2: Check a Single Student

  1. Click Student View
  2. Use the filter dropdown to select a student by Core ID
  3. Click Load Student Data
  4. Review all assessment tasks and completion status
  5. Check for missing data (shown in grey or with warning icons)
  6. Verify termination flags match calculations
[Screenshot: Student detail view showing task completion grid]

üí° Status Indicators

  • ‚Ä¢ ‚úÖ Green checkmark - Data verified, calculation matches
  • ‚Ä¢ ‚ö†Ô∏è Orange warning - Mismatch detected, needs review
  • ‚Ä¢ üî¥ Red highlight - Assessment terminated
  • ‚Ä¢ ‚ö™ Grey circle - Not started or missing data

Step 4.3: Check a Whole Class

  1. From the home page, click Class View
  2. Select a school from the dropdown
  3. Select a class from the available classes
  4. View the class completion heatmap showing all students
  5. Identify students with incomplete assessments (highlighted)
  6. Click on a student to see individual details
[Screenshot: Class view showing student grid with completion status]

Step 4.4: Check a Whole School

  1. Click School View
  2. Select a school from the dropdown
  3. View aggregated statistics for all classes in the school
  4. See completion percentages by task type
  5. Click on a class to drill down to class-level details
[Screenshot: School view showing class-level summary statistics]

Step 4.5: Check a Whole Group

  1. Click Group View
  2. Select a project group (1, 2, 3, 4, or 5)
  3. View all schools in that group
  4. Compare completion rates across schools
  5. Click on a school to see more details

Step 4.6: Check Entire District

  1. Click District View
  2. Select a district (Shatin, Sham Shui Po, etc.)
  3. View geographic aggregation of all schools in that district
  4. Track overall progress at the highest level
  5. Drill down to individual schools for details
[Screenshot: District view showing regional summary]

‚úÖ Navigation Tip: You can drill down from District ‚Üí Group ‚Üí School ‚Üí Class ‚Üí Student, or jump directly to any level from the home page.

5 Building the Cache

Build the cache to improve performance when loading large datasets in the Checking System.

Step 5.1: When to Build Cache

Build the cache:

  • After uploading a large batch of new assessments
  • When the Checking System feels slow to load
  • At the start of each day if you're doing heavy checking work
  • After any major data updates

Step 5.2: Build the Cache

  1. Open the Checking System
  2. Look for the Build Cache or Refresh Data button
  3. Click the button to start the cache building process
  4. Wait for the progress indicator to complete (may take 30-60 seconds)
  5. You'll see a success message when done

üí° What is Cache? The cache stores pre-processed data in your browser for faster loading. It's temporary and rebuilds automatically when needed, but manual refresh ensures you have the latest data.

[Screenshot: Cache building progress indicator]

‚úÖ Performance Boost: With cache built, loading student data is 5-10x faster!

6 Using Filters

Use filters to quickly find specific students, schools, or data matching certain criteria.

Step 6.1: Filter by Student

  1. Go to any view in the Checking System
  2. Find the Filter by Student dropdown
  3. Start typing the student's Core ID or name
  4. Select the student from the filtered list
  5. Click Apply Filter to show only that student's data
[Screenshot: Filter dropdown showing student search]

Step 6.2: Filter by School

  1. Use the Filter by School dropdown
  2. Select a school from the list
  3. The view updates to show only students from that school
  4. Use additional filters to narrow down further (e.g., by class or grade)

Step 6.3: Filter by Completion Status

  1. Look for the Show or Status filter
  2. Select from options like:
    • All students
    • Missing data only
    • Complete only
    • Incomplete only
  3. The table updates to show only matching records

üí° Combining Filters: You can combine multiple filters (e.g., School + Grade + Missing data) to pinpoint exactly what you need to review.

Step 6.4: Export Filtered Data

  1. Apply your desired filters
  2. Click the Export button (usually shows as Excel or CSV icon)
  3. Choose the export format (Excel recommended)
  4. The exported file includes only the filtered records
  5. Open the file to review or share the data
[Screenshot: Export options dialog]

‚úÖ Export Includes: All exports include a calculated Status Light column (üü¢ Complete, üî¥ Incomplete, üü° Post-Term, ‚ö™ Not Started) for easy quality checks.

Step 6.5: Clear Filters

  1. To reset and see all data, click Clear Filters or Reset
  2. All dropdowns return to their default state
  3. The full dataset is displayed again
